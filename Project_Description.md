## 三、项目内容：基于FPGA的饮料售货机

### 3.1 项目概述

设计并实现一个基于FPGA的饮料售货机系统。系统需支持销售模式与管理模式两类核心工作流程，并通过开发板外设（按键、拨码开关、LED、数码管）实现人机交互。

**系统重点能力如下：**

1. 用户可查看在售饮料信息与价格，选择饮料并支付后完成出货。
2. 出货后通过流水灯提示用户取货，若超时未取货需执行回收与退款。
3. 管理员可查看库存与销售金额，调整价格、补货、上下架饮料。
4. （Bonus）管理模式加入密码验证机制，连续3次输入错误触发报警。

## 四、基础功能（80分）

### 4.1 模式与输入输出

#### 4.1.1 模式定义

系统至少包含以下3种状态：

1. **主菜单状态**：选择进入销售模式或管理模式。
2. **销售模式**：面向普通用户完成饮料购买流程。
3. **管理模式**：面向管理员完成参数维护与状态查看。

#### 4.1.2 输入

输入方式由开发者自行设计，可使用按键 + 拨码开关、矩阵键盘、串口输入等。建议至少实现：

1. 菜单选择输入（模式切换、饮料选择、功能确认）
2. 数值输入（金额、价格、库存增量、密码）

#### 4.1.3 输出

系统输出建议分层如下：

1. **主界面/状态显示**：显示在售饮料列表、单价、库存、购买结果、管理信息
2. **LED指示**：状态灯与流水灯（提示可取货）
3. **数码管**：显示金额、倒计时、错误码或关键状态编号

### 4.2 销售模式

#### 4.2.1 商品展示

用户进入销售模式后，系统应展示当前可售饮料信息，包括：

1. 饮料编号（如1,2,3...）
2. 饮料名称（如可乐、雪碧、橙汁等，可英文简写）
3. 当前单价（单位可定义为元或角）
4. 库存状态（可显示库存数或"有货/缺货"）

#### 4.2.2 购买流程

购买流程至少包含以下步骤：

1. 用户选择饮料编号
2. 系统检查该饮料是否在售且库存大于0
3. 用户完成支付（可简化为余额输入 + 扣款）
4. 支付成功后系统出货，并将对应库存减1
5. 系统点亮流水灯，提示用户取货

#### 4.2.3 超时取货处理

出货后启动5秒取货计时：

1. 若用户在5秒内取走饮料，订单完成并记录销售金额
2. 若超过5秒未取走，系统执行饮料回收并退款，且本次销售金额不计入实收

#### 4.2.4 异常处理

至少处理以下异常：

1. 选择无效编号
2. 选择了已停售饮料
3. 库存不足
4. 支付金额不足

异常发生时需给出可视化提示（如不同的LED组合）。

### 4.3 管理模式

#### 4.3.1 状态查看

管理员进入管理模式后可查看：

1. 各饮料种类及库存数量
2. 每种饮料当前单价
3. 系统累计实收金额（已完成销售总额）
4. 当前停售列表（如有）

#### 4.3.2 参数维护

管理员可执行以下操作：

1. **修改价格**：调整指定饮料单价
2. **补货**：增加指定饮料库存
3. **停售/恢复售卖**：切换指定饮料的销售状态

#### 4.3.3 数据一致性要求

管理模式下的修改应实时生效，并影响销售模式行为：

1. 被停售饮料在销售模式不可购买
2. 价格调整后，新订单按新价格执行
3. 补货后可立即参与销售
