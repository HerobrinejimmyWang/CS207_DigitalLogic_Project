# EGO1 饮料售货机模块输入输出变量表 v5.1（中文修正版）

左栏为输入变量，右栏为输出变量；不包含 XDC pin 约束。

## 说明

本文件专门列出每个模块的输入/输出变量。表格左栏为输入变量，右栏为输出变量。变量名是建议接口名，队员实现时应尽量保持一致；若需要改名，必须同步修改顶层连接、testbench 和 Codex prompt。

本版修正了矩阵键盘输入链路命名：`matrix_keypad_scanner` 输出 `scan_key_valid/scan_key_row/scan_key_col`，`key_decoder` 输入同名信号；`key_decoder` 输出 `key_valid/key_code`，`input_event_router` 输入同名信号。

## 全局约定

| 类别 | 约定 |
| --- | --- |
| 时钟 | 除非明确说明，所有时序模块使用同一个 clk |
| 复位 | 内部 rst 统一高有效，由 reset_sync 产生 |
| 事件 | event_valid 为一周期脉冲；event_type/event_value 同周期有效 |
| 数据写请求 | *_req 建议为一周期脉冲，data_manager 在该周期采样 item/value |
| UI | 输出模块只读取 ui_snapshot 或派生 UI 信号，不修改业务状态 |
| 矩阵键盘链路 | scanner 输出 scan_key_*；decoder 输入 scan_key_*，输出 key_valid/key_code；router 输入 key_valid/key_code |

## reset_sync

把外部 reset 同步到系统时钟域，并输出内部统一高有效 rst。

| 输入变量 | 输出变量 |
| --- | --- |
| **clk** (input, 1)<br>系统时钟 | **rst** (output, 1)<br>内部统一复位，高有效 |
| **sys_rst_n** (input, 1)<br>外部 reset 输入，默认低有效；若上板发现极性相反，只改本模块 |  |

## matrix_keypad_scanner

扫描 4x4 矩阵键盘，包含消抖、长按锁定、释放检测。

| 输入变量 | 输出变量 |
| --- | --- |
| **clk** (input, 1)<br>系统时钟 | **row_active** (output, [3:0])<br>当前扫描行，一位为 1；顶层将对应 row 拉低，其余高阻 |
| **rst** (input, 1)<br>内部复位 | **scan_key_valid** (output, 1)<br>消抖后的一周期按键有效脉冲，直接连接到 key_decoder.scan_key_valid |
| **tick_1ms** (input, 1)<br>扫描节拍 | **scan_key_row** (output, [1:0])<br>稳定按键所在行，直接连接到 key_decoder.scan_key_row |
| **col_in** (input, [3:0])<br>列输入，列线由 XDC PULLUP，上拉后未按为 1，按下为 0 | **scan_key_col** (output, [1:0])<br>稳定按键所在列，直接连接到 key_decoder.scan_key_col |

## key_decoder

把 key_row/key_col 转为统一 key_code。

| 输入变量 | 输出变量 |
| --- | --- |
| **scan_key_valid** (input, 1)<br>来自 matrix_keypad_scanner.scan_key_valid 的按键有效脉冲 | **key_valid** (output, 1)<br>译码后按键有效脉冲，直接连接到 input_event_router.key_valid |
| **scan_key_row** (input, [1:0])<br>来自 matrix_keypad_scanner.scan_key_row 的按键行 | **key_code** (output, [3:0])<br>0-9 为数字；A=4'hA，B=4'hB，C=4'hC，D=4'hD，*=4'hE，#=4'hF，直接连接到 input_event_router.key_code |
| **scan_key_col** (input, [1:0])<br>来自 matrix_keypad_scanner.scan_key_col 的按键列 |  |

## input_event_router

把具体 key_code 转成抽象事件。注意：不判断当前状态是否允许。

| 输入变量 | 输出变量 |
| --- | --- |
| **key_valid** (input, 1)<br>来自 key_decoder.key_valid 的译码后按键有效脉冲 | **event_valid** (output, 1)<br>抽象事件有效脉冲 |
| **key_code** (input, [3:0])<br>来自 key_decoder.key_code 的具体按键编码 | **event_type** (output, [2:0])<br>EV_DIGIT/EV_PREV/EV_BACK/EV_HOME/EV_NEXT/EV_CLEAR/EV_CONFIRM |
|  | **event_value** (output, [3:0])<br>数字事件的数值 0-9，其它事件可为 0 |
